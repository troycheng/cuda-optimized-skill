# CUDA Kernel Optimizer Skill v2.1 Implementation Plan

> **For the AI agent implementing this plan:** Required sub-skill: use
> `superpowers:executing-plans` to implement each task in order. Track progress
> with the checkboxes below.

**Goal:** Make the CUDA/CUTLASS/Triton optimizer skill discoverable, portable,
and measurement-correct against the July 2026 CUDA ecosystem.

**Architecture:** Keep the existing orchestration and state model, but isolate
hardware-dependent operations behind small helpers that can be tested with the
standard library. Treat profiling and bottleneck classification as evidence
pipelines: mark the exact target launch, preserve raw metrics, and report when
the available evidence is heuristic rather than authoritative.

**Technical stack:** Python 3.10+, `unittest`, PyTorch CUDA runtime, Nsight
Compute CLI 2026.2, Triton 3.7, CUTLASS 4.6, Codex skill metadata.

---

```python
def _time_iterations(fn, warmup: int, repeat: int, cuda=None) -> list[float]:
    cuda = cuda or _require_torch().cuda
    for _ in range(warmup):
        fn()
    cuda.synchronize()
    samples = []
    for _ in range(repeat):
        start = cuda.Event(enable_timing=True)
        end = cuda.Event(enable_timing=True)
        start.record()
        fn()
        end.record()
        end.synchronize()
        samples.append(float(start.elapsed_time(end)))
    return samples
```

Reject non-positive `warmup` or `repeat` values with a CLI error and keep the
existing JSON keys for average, median, minimum, and maximum.
